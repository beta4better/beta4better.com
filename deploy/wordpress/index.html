让人又爱又恨的FreeBSD

这两天在拼命地折腾 FreeBSD，其实主要还是在 Intel 的集成显卡的驱动上，用 VESA 来驱动的话不是那么理想。

笔记本上1280x800的分辨率是正常的，不过逐行刷新卡的要命。而在台式机上 HD Graphics 的显卡分辨貌似只能到1280x1024，在最佳分辨率是1680x1050的显示器上字体发虚，觉得很别扭。

好在现在社区提供一种方法，用 WITH_NEW_ORG 和 WITH_KMS 重新编译整个系统和 xf86-video-intel，这样分辨率正常，不过不能在控制台和图形界面之间切换，启动图形界面后要切换到控制台就会花屏，而直接退出图形界面的话会直接黑屏，暂时无解。

早饭，大家都知道是很重要的，一天中最重要的一顿饭了。

早饭要吃好，午饭要吃饱，晚饭要吃少。

我也在坚持吃早饭了，最近胃一直不太好，是不是的会难受下。不是很难受，稍微吃点健胃消食片就会好些。喝过一段时间的养乐多，也挺管用的，不过就是太贵了，十块钱一板，喝不了几天就没了，或许还是按时吃饭，慢慢养的好。

  • 鸡蛋糕，两块钱可以买四个，小个的那种

关于VirutalBox的四种网络设置

  • NAT：网络地址转换（Network Address Translation），应该是虚拟机要将数据包交给主机，再由主机的的网卡发出去

  • Birdged Network：桥接，这种方式，就是虚拟机与主机平等平坐，两个相当于独立机子，外面的内网里也可以访问我的虚拟机

  • Internal Network：内部网络（可以是虚拟机与虚拟机之间）

  • Host-Only：只与主机通信

第一种、第二种、第四种我都尝试过了，确实如上所说。

实在家里实验的，有路由器，给虚拟机开三个网卡就可以了，如果不进行第二种设置，除了主机外其他的电脑访问不了虚拟机。

在VirtualBox里设置好了后还在要配置文件：/etc/network/interfaces里把相应的网卡都添加进去：

allow-hotplug eth0

iface eth0 inet dhcp

allow-hotplug eth1

iface eth1 inet dhcp

allow-hotplug eth2

iface eth2 inet dhcp

图省事全部用的DHCP。

VirtualBox虚拟机迁移后重新启用网卡

Ubuntu 11.04 下给Firefox安装Flash插件

在Ubuntu下安装Flash插件貌似相当方便了，在Firefox下打开Adobe Flash Player的网站：

http://get.adobe.com/flashplayer/

在下拉框里悬在APT for Ubuntu

再点击Download Now

然后会直接打开Ubuntu Software Center

点击Install

安装完成后重新启动Firefox就可一播放视频了。
