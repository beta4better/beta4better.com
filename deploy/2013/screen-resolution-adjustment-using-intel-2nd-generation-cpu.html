Debian下设置正确的屏幕分辨率

2013-01-17 10:49:00  beta4better  comments  permlink 

起因如下，在一台U3400处理器的笔记本上装的 Debian Wheezy 屏幕分辨率1280x800，正常显示；但是在2012年刚买的一台准系统上，配的处理器是I5-2320，装完 Debian Squeeze 后显示器的分辨率一直达不到1980x1650，看着很别扭，升级成 Wheezy后依旧不行……

同一台机器，之前在安装 FreeBSD 的时候也是有这样的问题，屏幕分辨率不正确，要重新编译内核才行。

一开始的时候我觉得会是 Squeeze 的版本太低，于是就升级成了 Wheezy，但屏幕看上去依旧是扁扁的。联想到之前用 FreeBSD 的情况，我觉得应该是驱动程序的问题，有 intel 的驱动不使用，而是在用 VESA 的通用设备驱动。于是我扁把 VESA 的通用驱动删除，重启。果真启动 X 就黑屏了。

至此位置，正确的笔记本和不正常的准系统上配置不一样的就是内核了。 笔记本上的内核是3.2，虽然2.6并存但是如果用2.6启动的话起不来。 于是乎我便把准系统上的内核也安装了个3.2的内核。 重启一切问题解决。系统自动配用 intel 的驱动，一切OK。

记下来已被后查。
